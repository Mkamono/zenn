# Zenn記事の書き方ガイド

## フロントマター (メタデータ)

記事の冒頭に以下の形式でメタデータを記述する：

```yaml
---
title: "記事のタイトル"
emoji: "🌏"  # 1つの絵文字
type: "tech"  # tech: 技術記事 / idea: アイデア
topics: ["googlecloud", "cloudrun"]  # タグの配列
published: true  # 公開状態: true/false
---
```

### 各項目の説明
- **title**: 記事タイトル。読みやすく、内容を的確に表現する
- **emoji**: 記事を象徴する絵文字1つ
- **type**:
  - `tech`: 技術記事
  - `idea`: アイデア記事
- **topics**: 関連するタグの配列。適切なタグを選択し、5個以内に収める
- **published**:
  - `true`: 公開状態
  - `false`: 下書き状態

## 記事構造

### 1. はじめに
記事の背景、目的、読者への価値を明確に示す。問題意識や記事を書くきっかけを説明。

### 2. 本文
- 見出しを適切に使い分ける（##、###など）
- 具体例や図表、コードブロックを活用
- 読者が理解しやすい論理的な構成にする

### 3. まとめ
記事の要点を整理し、読者が次に取るべきアクションや参考情報を提示。

## 文章スタイル

### 基本ルール
- 文末に句点（。）を付ける
- 箇条書きは「-」を使用
- コードブロックには適切な言語指定を行う

### 強調・装飾
- **太字**: 重要なポイント
- `コード`: 技術的な用語やコマンド
- リンク: 関連する外部リソースへの参照

## 特殊記法

### メッセージボックス
```
:::message
重要な注意事項や補足情報
:::
```

### 画像の挿入
```
![alt_text](/images/folder_name/image_name.png)
```
- 画像は `/images/` ディレクトリに配置
- 記事ごとにサブディレクトリを作成することを推奨

## コンテンツのポイント

### 技術記事の場合
- 実践的な内容を重視
- 実際のコード例や設定例を含める
- トレードオフや制約も明記
- 読者が実際に試せる具体的な手順を提供

### 構成のベストプラクティス
- 問題提起 → 解決策 → 実装例 → 注意点 → まとめ の流れ
- 読者の時間を尊重し、要点を明確に
- 根拠となるデータや公式ドキュメントへのリンクを含める

## 品質向上のポイント

1. **読者目線**: 読者が求める情報を最優先に
2. **具体性**: 抽象的な説明よりも具体例を重視
3. **実用性**: 実際に活用できる内容を心がける
4. **正確性**: 情報の裏付けを取り、最新の情報を提供
5. **構造化**: 見出しや箇条書きで読みやすく整理
